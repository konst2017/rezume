«Добрый день.
Посмотрели сайт http://konst2020.qlihost.ru/testzad3/
И код тестовой задачи https://github.com/konst2017/rezume/tree/main/testzad3.
По сайту основные замечания:
- Верстка http://prntscr.com/w347cr должна соответствовать дизайну http://prntscr.com/w346i1. Это важное требование.
- Не всегда работает фильтрация. Например, если задать только "начальный возраст" и не задавать "конечный возраст", 
то фильтрация не работает. Информация в полях фильтра не сохраняется при поиске. Так же сбрасывается фильтр при 
переходе на следующую страницу в блоке пагинации.
По коду основные замечания:
- Код должен быть отформатирован в соответствии со стандартом psr-12. https://www.php-fig.org/psr/psr-12/ 
(или на русском https://
redwerk.jobs/blog/psr-12-%D0%BF%D0%B5%D1%80%D0%B5%D0%B2%D0%BE%D0%B4-%D1%80%D0%B0%D1%81%D1%88%D0%B8%D1%80%D0%B5%D0%BD%D0%BD%D0%BE%D0%B3%D0%BE-%D1%81%D1%82%D0%B0%D0%BD%D0%B4%D0%B0%D1%80%D1%82%D0%B0-%D0%BE%D1%84%D0%BE/).
 Например не использовать табуляцию для отступа, а использовать 4 пробела, для каждого уровня кода. 
Не добавлять пробелы в конце строки.
- Именование переменных, классов, методов должно отражать их суть. Например, 
в коде встречается слово "Product". В тестовой задаче есть рюземе. 
Правильно было бы использовать слово "Resume".
- Так же при именовании переменных не нужно использовать транслитерацию. 
Например в коде есть "wozr". Вместо этого именования нужно использовать "age".
- Для создания таблиц в базе данных нужно использовать миграции.
 https://yiiframework.com.ua/ru/doc/guide/2/db-migrations/
- Для наполнения таблиц тестовыми данными удобнее использовать фикстуры.
 https://yiiframework.com.ua/ru/doc/guide/2/test-fixtures/
- В репозиторий не нужно коммитить папку vendor.
- Не нужно коммитить закомментированный код.
 Например, frontend/controllers/PoiskController.php:40 // 'class' => 'yii\captcha\CaptchaAction',»
